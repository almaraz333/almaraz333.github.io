Sports Web Scraper



A Python program to gather data on every professional sports player in California from 1965 to now.




Project Overview


Challenge:

Design and code an automated web scraper to gather the name, team, date of birth, and EAMS claim status on over 30,000 individuals.

Team:

Myself


The Process



What is the Goal?

The goal for this project was to create a program to effectively automate a process that would otherwise take weeks to complete. The amount of data this program would have to go though would be immense. Not only would this program have to be able to search through tens of thousands of people across multiple sites, but it would have to do it efficiently due to time constraints.




Necessities:


  • Fast
  • Flexible
  • Automated
  • Reusable
  • Readable and parsable output
  • Efficient code to reduce runtime


The Challenge of Multiple Sites

In the research stage of the project I realized I would need to use multiple sites to gather all the data that was needed. This brought up the challenge of creating a flexible program that could navigate and handle the different architectures of each site. Due to this there were very specific restraints on the code. After much testing and retesting I found a way to create a program that could search through different sites and pick up on which people it needed to get information for.




What I Learned

  1. Flexibility is key. When creating programs it is important that the code you write is flexible. There is no way of knowing what kind of problems could arise in real-world situations. This is why your code must be flexible enough to handle any problems that could happen or at the very least be able inform the developer there is a problem so they can fix it.

  2. In order to provide a good product and experience you need to keep your client in mind. In this case it was a businessperson who needed the data to be easily readable and accessible. For this to be the case the output must be formatted correctly. By this I mean taking hundreds of lists, arrays, and key-value pairs and turning them into a readable file with meaningful categories.

  3. Being able to manage multiple tasks is a huge part of being a professional. This project was long and required a lot of attention but could not impact the quality of other projects I was working on. Taking time to create a good product while also leaving time to make sure other priorities aren't impacted is essential in a fast-paced environment like today. Being able to balance these tasks is a skill I am proud of.

Deliverables

The final deliverable were multiple CSV files based on the team. Due to the size of the data, some file being thousands of lines long, CSV was the best choice for readable, searchable data that could easily be exported.
