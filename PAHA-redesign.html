Redesign of the Professional Athletes Healthcare Advocates Site



This project was a redesign of the PAHA.com website. Because of who this company interacted with it was extremely important that the site was easy to use and mobile accessible. This meant creating a site that adhered to the ADA & WCAG guidelines for accessibility.




Project Overview


Challenge:

Create an accessible site that encourages the user to contact the company while working around unfamiliar code all while working remotely.



Team:

Myself


The Process



What is the Goal?

The goal was to design an accessible site that let the user easily find the information they needed in order to know if they were eligible to make a claim as well as contact the company to schedule and appointment. All of this had to be done in an accessible way according to the ADA.




Necessities:


  • Accessible
  • ADA & WCAG Compliant
  • User Friendly
  • Easy to Navigate
  • Multiple Call to Actions


The Importance of Accessibility

When you are creating something from the ground up it is easy to make it accessible. You know beforehand exactly which components need to have which properties. But when you have to retrofit a site to be more accessible it becomes slightly more difficult. Not necessarily hard, but going back in after something is done, finding which parts need to change, and them going through unfamiliar code to exact those changes can be a problem. Not only is accessibility important legally, but to me as a person it is important morally. Making sure no one is excluded from an experience should be the top priority of any coder or designer.




What I Learned

  1. Accessibility is key. I already knew that accessibility was extremely important but this really cemented that fact for me. I was able to really picture who the uer was and how accessibility would impact them. For me, a site being accessible is not only a necessity but a moral obligation.

  2. Working with someone else's code can be frustrating. Even though it's pretty common to have to work around unfamiliar code it still can be frustrating sometimes. This is why I value having a solid understanding of the code your're working with so much. Having this understanding makes unfamiliar code much easier to understand.

  3. Communication is important. Working with a client remotely ahs its challenges. To work through these challenges is important and to do this communication is key. If there is solid communication with the client then working remotely is no problem at all.
